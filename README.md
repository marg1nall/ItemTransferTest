## Описание

Тестовое задание: реализация доски, инвентаря, предметов и системы сохранения.

## Используемые технологии

- **Unity 6000.3.10f1**
- **Zenject**
- **Паттерн Factory**
- **Task**

## Архитектура

В проекте использован подход резделения доменов на классы. Внутри кода выделены папки:

- `Api` – контракты, интерфейсы и абстракции.
- `Logic` – бизнес-логика, правила работы с предметами и инвентарём.
- `Models` – данные и модели предметов/ячеек.
- `UI` – отображение, контроллеры окон, взаимодействие с пользователем.

Решение о таком подходе сделал исходя из следующих пунктов:

1. В тз не было обозначена желаемая архитектура
2. Скорость разработки, то есть вместо продумывания архитектуры наперед я пользовался, тем что у нас есть определенный useCase, Model

В общем такой подход не разивал спагетти код и позволял эффективно разрабывать систему
