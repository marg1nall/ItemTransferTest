# Item Transfer Test Assignment


![Анимация](https://github.com/user-attachments/assets/26d749b4-d5d1-4c81-8019-3a66dd5b8947)


## Описание

Тестовое задание: реализация доски, инвентаря, предметов и системы сохранения.

## Используемые технологии

- **Unity 6000.3.10f1**
- **Zenject**
- **Паттерн Factory**
- **Task**

## Архитектура

Используется формат 16:9 
В проекте использован подход резделения доменов на классы. Внутри кода выделены папки:

- `Api` – контракты, интерфейсы и абстракции.
- `Logic` – бизнес-логика, правила работы с предметами и инвентарём.
- `Models` – данные и модели предметов/ячеек.
- `UI` – отображение.

Решение о таком подходе сделал исходя из следующих пунктов:

1. В ТЗ не была указана предпочтительная архитектура, поэтому я выбрал подход, который показался мне наиболее понятным и гибким для данной задачи.
2. Скорость разработки, то есть вместо долгого проектирования я опирался на use cases и модели данных с представлением. 

В общем такой подход не развивал спагетти код и позволял эффективно разрабывать систему
